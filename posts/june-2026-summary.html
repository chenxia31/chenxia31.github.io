2026年6月总结 — 读书记录·新鲜链接·运势

2026年的六月，北京的夏天热得早。这个月忙着拆解业务逻辑、打磨方法论，也在碎片时间里读了几本书、关注了一些有意思的 AI 新方向。整理成文，供日后回看。

📚 读书记录

1. 《骆驼祥子》— 老舍

读的是骆驼祥子，但看的不是车夫，看的是一个”超人哲学在现实中的失败”。

祥子过分迷信个人努力与个人成就——成功全靠自己的勤劳与汗水，失败便是自己的无能与懒惰。他把自己与其他车夫划清界限，不求助、不结社、不去银行存钱用存钱罐（结果被骗光）。

最致命的是：他曾经对他人的鄙夷，在失败后全部变成了骂自己的话。自己 PUA 自己，失去尊严后诞生出更膨胀的自尊。一旦发现自己坠落到了”那些车夫的阶级”，便毫无负担地放弃正直，全盘接受他们的行为模式，甚至更堕落。

这不是享乐主义的问题。这是把个人奋斗绝对化带来的认知陷阱——没有系统思维，没有向外求助的能力，一切归因于自己，成就时膨胀，失败时自毁。

2. 《人月神话》第 1 章「焦油坑」— 弗雷德里克·布鲁克斯

岸上的船儿，如同海上的灯塔，无法移动。——荷兰谚语

布鲁克斯用”焦油坑”比喻大型软件开发——巨兽挣扎得越猛烈，焦油纠缠得越紧。

核心框架：编程系统产品成本是独立程序的 9 倍

  • 程序 → 编程产品：×3（通用化、测试、文档）
  • 程序 → 编程系统构件：×3（接口、资源限制、组合测试）
  • 编程系统产品：×9（两者兼具）

编程的五重乐趣：

  1. 创建事物的纯粹快乐
  2. 开发对他人有用的东西
  3. 零部件组装精密运行的魔术感
  4. 工作的非重复性带来的持续学习
  5. 代码介质的灵活可控

编程的五大苦恼：

  1. 必须追求完美——计算机不宽容任何错误
  2. 权威与责任不匹配——目标由他人设定
  3. 依赖他人的低质量代码
  4. 找 bug 的重复性（调试呈线性甚至二次方复杂度）
  5. 产品完成时可能已经过时

对许多人而言，乐趣远大于苦恼。

🔗 新鲜链接 & 有意思的东西

AI Agent 方向

1. 微软 SkillOpt 框架 — 像训练神经网络一样训练 Agent Skill

来源： Vincent (@vincentlogic)

微软联合上海交大、同济、复旦发布 SkillOpt 框架。核心思路：不动模型权重，训练的是 Skill 文件（自然语言 prompt 和指导文档）。

机制：执行任务 → 记录过程 → 复盘 → 修改 Skill → 验证效果，自动闭环迭代。

实验结果夸张：7个大模型、6类任务、3种 Agent 环境，52 组测试全部第一或并列第一。最有价值的是跨模型迁移能力：

  • GPT-5.4 优化好的 skill → GPT-5.4-nano，提升 5.6 分
  • Claude Code 优化好的 skill → Codex，提升 29.4 分
  • 一个 best_skill.md 文件，到处能用

对 Hermes Agent 的 self-evolution 方向很有参考价值。

🔗 https://x.com/vincentlogic/status/2066458456564080965

2. 设计循环，而不是写提示词

来源： @yanhua1010

继 Prompt Engineering 之后的升级思考：人不再写提示词，转而去设计 Agent 运行的循环（Loop）。X 上开始出现新概念 “Agent Loop”。

3. Awesome Agent Loops — 精选实战命令合集

来源： @qingq77

收录 Claude Code 和 Codex 用户分享的最佳 /loop、/goal、/schedule 实战命令，可直接复制使用。

🔗 https://github.com/serenakeyitan/awesome-agent-loops

4. Hyper-Extract — 文档转知识图谱 + 时间轴

来源： @AIExplorerTim

用 LLM 将非结构化文本转为结构化知识图谱、超图、时空提取，一条命令搞定。

🔗 https://github.com/yifanfeng97/Hyper-Extract

5. Agent Reach — 全网自动搜索 Agent

来源： @aronhouyu

开源 Agent，支持 B站、YouTube、Reddit、RSS、GitHub、X、领英等 17 个平台，零配置可用 8 个渠道。（已安装 ✅）

🔗 https://github.com/Panniantong/agent-reach

6. 人机交互进化论（自拟笔记）

远古（打孔纸带）→ 上古（汇编）→ 中古（CLI）→ 近古（GUI）→ 近代（触摸屏）→ AI 时代

到了 AI 时代，人在中间似乎没用了——机器与机器沟通不需要向人兼容。讽刺的是，MCP、Skill 等 AI 工具链反而在回归 CLI，有继续往上古回归的趋势。

🧠 这个月的思考碎片

关于 “Must”（绝对使命感）

一段关于存在主义危机的思考：人最核心的动力是寻找意义。”Must”不是被迫的琐事，而是一种不计得失、超越本能的内心召唤——消防员冲进火场、母亲保护孩子，做这些事时没有权衡利弊，只有”我非做不可”。

真正的使命感不能靠外界强加，而是在经历挫折、爱、失去与成长后，像植物生根一样从内心长出来的信仰。与弗兰克尔《活出意义来》共鸣。

拥有 “Must” 就是有了承受一切苦难和无聊的底气。

关于 “大脑的时代”

从今往后，是大脑的时代，也就是知识与想法创造价值的时代。

知识与想法正在取代传统生产要素成为核心价值来源。与 “焦油坑” 形成奇妙对照——软件工程是让人挣扎的泥潭，但知识和想法是唯一能跨越它的桥梁。

关于绩效与成长（来自工作笔记）

六月在业务上继续打磨方法论沉淀的重要性。核心体会：

  • 业务判断力是数据科学 2-3 年时拉开差距的核心
  • 方法论沉淀——把做过的事变成可复用的资产
  • 技术纵深——不要均匀用力，选一个方向打深
  • 可见度——没被看见的分析等于没做
